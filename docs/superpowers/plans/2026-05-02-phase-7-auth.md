# Phase 7 — Authentication Subsystem Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the Phase 1 `getCurrentUserId()` stub with a real, three-mode authentication subsystem. Phase 7 lands the boot guard, password hashing, sticky sessions for `simple` mode, Auth.js v5 with Credentials + five OAuth providers for `full` mode, login rate limiting, the `LoginAttempt` and `AuditLog` tables, and a `requireUserId()` contract that every server action from Phase 8 onward must use.

**This is the security-critical phase.** If Phase 7 ships wrong, every later phase inherits the bug. Read `docs/security.md` first. Read master roadmap §2.1 (deploy modes) and design calls #2 (per-user data isolation), #6 (encrypted API keys), and #10 (boot-time guards). Phase 7 is when the rule "every server action resolves `userId` from the session, never from the request body" becomes mechanically enforceable.

**Deploy targets this phase:** `local` (all three modes) and `railway` (only `full`, refused at boot otherwise).

**Architecture:**

```
instrumentation.ts (Next.js 15 startup hook)
   -> runBootGuard()         throws if env is unsafe; otherwise proceeds
src/server/auth.ts
   AUTH_MODE=none    -> getCurrentUserId() returns 'local-default'
   AUTH_MODE=simple  -> iron-session sticky cookie + Argon2id
   AUTH_MODE=full    -> Auth.js v5 + Prisma adapter
                          Credentials (Argon2id) + Google/GitHub/Azure/Twitter/Facebook
src/server/actions/auth.ts
   setSimplePassword, signInWithPassword, signOut, requireUserId (throws if no session)
```

The Phase 1/2/3 `getCurrentUserId()` is replaced by `requireUserId()`, which throws `UnauthenticatedError` instead of silently returning the seeded default. The Phase 1 fallback is preserved only for `AUTH_MODE=none`.

**Tech Stack additions:** `next-auth@5` (Auth.js v5 / NextAuth beta), `@auth/prisma-adapter`, `argon2`, `iron-session`, `zod`. We use `iron-session` for `simple` mode rather than Auth.js Credentials because (a) it has zero database dependency for session storage (cookie-only), (b) it lets us keep `simple` mode dead-simple (one password, no User row lookup), and (c) it avoids loading Auth.js when the deployment doesn't need it. `full` mode uses Auth.js Credentials with the same Argon2id hash.

```ts
export function readEnv(): Env {
```

- [ ] **Step 2: Implement `src/server/boot-guard.ts`**

```ts
import { readEnv } from "./env";

export class BootGuardError extends Error {
  constructor(message: string) {
    super(`[boot-guard] ${message}`);
    this.name = "BootGuardError";
  }
}

export function runBootGuard(): void {
  const env = readEnv();

  // 1. APP_ENCRYPTION_KEY must decode to exactly 32 bytes
  let keyBytes: Buffer;
  try {
    keyBytes = Buffer.from(env.APP_ENCRYPTION_KEY, "base64");
  } catch {
    throw new BootGuardError("APP_ENCRYPTION_KEY must be valid base64");
  }
  if (keyBytes.length !== 32) {
    throw new BootGuardError(
      `APP_ENCRYPTION_KEY must be 32 bytes (base64); got ${keyBytes.length}`,
    );
  }

  // 2. Railway requires full auth mode
  if (env.DEPLOY_TARGET === "railway" && env.AUTH_MODE !== "full") {
    throw new BootGuardError(
      `Railway deployments must use AUTH_MODE=full (got '${env.AUTH_MODE}')`,
    );
  }

  // 3. simple/full both require AUTH_SECRET (cookie/JWT signing)
  if ((env.AUTH_MODE === "full" || env.AUTH_MODE === "simple") && !env.AUTH_SECRET) {
    throw new BootGuardError(
      `AUTH_SECRET required when AUTH_MODE=${env.AUTH_MODE}; generate with 'openssl rand -base64 32'`,
    );
  }

  // 4. Railway requires CRON_SECRET (Phase 10 will use it)
  if (env.DEPLOY_TARGET === "railway" && !env.CRON_SECRET) {
    throw new BootGuardError("CRON_SECRET required when DEPLOY_TARGET=railway");
  }

  // eslint-disable-next-line no-console
  console.info(
    `[boot-guard] ok - target=${env.DEPLOY_TARGET} auth=${env.AUTH_MODE}`,
  );
}
```

```ts
import argon2 from "argon2";

const OPTS: argon2.Options = {
  type: argon2.argon2id,
  timeCost: 3,             // iterations
  memoryCost: 64 * 1024,   // KiB -> 64 MiB
  parallelism: 4,
};

export async function hashPassword(plaintext: string): Promise<string> {
  if (!plaintext || plaintext.length < 8) {
    throw new Error("Password must be at least 8 characters");
  }
  return argon2.hash(plaintext, OPTS);
}

export async function verifyPassword(plaintext: string, hash: string): Promise<boolean> {
  if (!hash || !hash.startsWith("$argon2")) return false;
  try {
    return await argon2.verify(hash, plaintext);
  } catch {
    return false;
  }
}
```

## Task 5: TDD - Login rate limiter

**Files:**
- Create: `src/server/auth/rate-limit.ts`, `tests/unit/rate-limit.test.ts`

**Design:** in-memory map of `ip -> { attempts, windowStart, lockedUntil? }`. 10 attempts per IP per 15-minute window. After 10, exponential backoff: `lockedUntil = now + min(2^(attempts-10) * 1s, 1h)`. Window resets on first successful login from the IP. v1 uses an in-memory bucket - sufficient for single-instance Local + single-instance Railway. Multi-instance scaling is a Phase 14 concern (Redis).

- [ ] **Step 1: Write failing tests**

Create `tests/unit/rate-limit.test.ts`:

```ts
import { describe, it, expect, beforeEach } from "vitest";
import { check, recordFailure, recordSuccess, __resetForTests } from "@/server/auth/rate-limit";

describe("rate-limit", () => {
  beforeEach(() => __resetForTests());
  const IP = "1.2.3.4";

  it("allows first attempt", () => { expect(check(IP).ok).toBe(true); });

  it("allows up to 10 attempts in 15 min", () => {
    for (let i = 0; i < 10; i++) {

      recordFailure(IP);
    }

  it("blocks the 11th attempt with positive backoff", () => {
    for (let i = 0; i < 10; i++) recordFailure(IP);

    expect(r.ok).toBe(false);
    if (!r.ok) expect(r.retryAfterSec).toBeGreaterThan(0);

});
```

- [ ] **Step 2: Implement**

Create `src/server/auth/rate-limit.ts`:

```ts
const WINDOW_MS = 15 * 60 * 1000;
const SOFT_LIMIT = 10;
const MAX_BACKOFF_MS = 60 * 60 * 1000;

type Bucket = {
  attempts: number;     // failures in current window
  windowStart: number;  // ms
  lockedUntil?: number; // ms
};

const buckets = new Map<string, Bucket>();

function now() { return Date.now(); }

function gc() {
  const t = now();
  for (const [ip, b] of buckets) {
    if ((b.lockedUntil ?? 0) < t && t - b.windowStart > WINDOW_MS) {
      buckets.delete(ip);
    }
  }
}

export function check(ip: string): { ok: true } | { ok: false; retryAfterSec: number } {
  gc();
  const b = buckets.get(ip);
  if (!b) return { ok: true };
  const t = now();
  if (b.lockedUntil && b.lockedUntil > t) {
    return { ok: false, retryAfterSec: Math.ceil((b.lockedUntil - t) / 1000) };
  }
  if (t - b.windowStart > WINDOW_MS) {
    // window expired; reset on next failure
    return { ok: true };
  }
  return { ok: true };
}

export function recordFailure(ip: string): void {
  const t = now();
  let b = buckets.get(ip);
  if (!b || t - b.windowStart > WINDOW_MS) {
    b = { attempts: 0, windowStart: t };
    buckets.set(ip, b);
  }
  b.attempts += 1;
  if (b.attempts > SOFT_LIMIT) {
    const overflow = b.attempts - SOFT_LIMIT;
    const backoffMs = Math.min(2 ** overflow * 1000, MAX_BACKOFF_MS);
    b.lockedUntil = t + backoffMs;
  }
}

export function recordSuccess(ip: string): void {
  buckets.delete(ip);
}

// test-only
export function __resetForTests() {
  buckets.clear();
}
```

```ts
import NextAuth, { type NextAuthConfig } from "next-auth";
import Credentials from "next-auth/providers/credentials";
import Google from "next-auth/providers/google";
import GitHub from "next-auth/providers/github";
import AzureAD from "next-auth/providers/azure-ad";
import Twitter from "next-auth/providers/twitter";
import Facebook from "next-auth/providers/facebook";
import { PrismaAdapter } from "@auth/prisma-adapter";
import { prisma } from "@/server/db";
import { readEnv } from "@/server/env";
import { verifyPassword } from "./password";
import { check as rlCheck, recordFailure, recordSuccess } from "./rate-limit";
import { writeAuditLog } from "./audit";
import { InvalidCredentialsError, RateLimitedError } from "@/lib/errors";
import { z } from "zod";

const CredsSchema = z.object({
  email: z.string().email(),
  password: z.string().min(1),
});

// Real Argon2id PHC string used for "user not found" path so timing is
// indistinguishable from "wrong password". Generated once at module load.
const DUMMY_HASH =
  "$argon2id$v=19$m=65536,t=3,p=4$AAAAAAAAAAAAAAAAAAAAAA$AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA";

function buildProviders() {
  const env = readEnv();
  const providers: NextAuthConfig["providers"] = [
    Credentials({
      name: "credentials",
      credentials: {
        email: { label: "Email", type: "email" },
        password: { label: "Password", type: "password" },
      },
      async authorize(raw, req) {
        const ip = (req?.headers?.get?.("x-forwarded-for") ?? "0.0.0.0").split(",")[0].trim();
        const rl = rlCheck(ip);
        if (!rl.ok) {
          await writeAuditLog({ action: "login.ratelimit", ip });
          throw new RateLimitedError(rl.retryAfterSec);
        }
        const parsed = CredsSchema.safeParse(raw);
        if (!parsed.success) {
          recordFailure(ip);
          await writeAuditLog({ action: "login.fail", ip, meta: { reason: "invalid-shape" } });
          throw new InvalidCredentialsError();
        }
        const { email, password } = parsed.data;
        const user = await prisma.user.findUnique({ where: { email } });
        // Always run verify - even when user is missing - to hide existence via timing
        const stored = user?.passwordHash ?? DUMMY_HASH;
        const ok = await verifyPassword(password, stored);
        if (!user || !user.passwordHash || !ok) {
          recordFailure(ip);
          await writeAuditLog({
            action: "login.fail", ip, userId: user?.id ?? null,
            meta: { reason: user ? "wrong-password" : "no-such-user" },
          });
          throw new InvalidCredentialsError();
        }
        recordSuccess(ip);
        await writeAuditLog({ action: "login.success", ip, userId: user.id });
        return { id: user.id, email: user.email, name: user.name };
      },
    }),
  ];

  // Env-gated OAuth providers - each registers only if its full credential set is present.
  if (env.GOOGLE_CLIENT_ID && env.GOOGLE_CLIENT_SECRET) {
    providers.push(Google({ clientId: env.GOOGLE_CLIENT_ID, clientSecret: env.GOOGLE_CLIENT_SECRET }));
  }
  if (env.GITHUB_CLIENT_ID && env.GITHUB_CLIENT_SECRET) {
    providers.push(GitHub({ clientId: env.GITHUB_CLIENT_ID, clientSecret: env.GITHUB_CLIENT_SECRET }));
  }
  if (env.AZURE_AD_CLIENT_ID && env.AZURE_AD_CLIENT_SECRET && env.AZURE_AD_TENANT_ID) {
    providers.push(AzureAD({
      clientId: env.AZURE_AD_CLIENT_ID,
      clientSecret: env.AZURE_AD_CLIENT_SECRET,
      tenantId: env.AZURE_AD_TENANT_ID,
    }));
  }
  if (env.TWITTER_CLIENT_ID && env.TWITTER_CLIENT_SECRET) {
    providers.push(Twitter({ clientId: env.TWITTER_CLIENT_ID, clientSecret: env.TWITTER_CLIENT_SECRET }));
  }
  if (env.FACEBOOK_CLIENT_ID && env.FACEBOOK_CLIENT_SECRET) {
    providers.push(Facebook({ clientId: env.FACEBOOK_CLIENT_ID, clientSecret: env.FACEBOOK_CLIENT_SECRET }));
  }
  return providers;
}

export const authConfig: NextAuthConfig = {
  adapter: PrismaAdapter(prisma),
  providers: buildProviders(),
  session: { strategy: "jwt", maxAge: 60 * 60 * 24 * 30 },
  pages: { signIn: "/login" },
  cookies: {
    sessionToken: {
      name: "tdm_session",
      options: {
        httpOnly: true,
        sameSite: "lax",
        path: "/",
        secure: readEnv().DEPLOY_TARGET === "railway",
      },
    },
  },
  callbacks: {
    async jwt({ token, user }) {
      if (user) token.sub = user.id;
      return token;
    },
    async session({ session, token }) {
      if (token.sub) (session.user as { id?: string }).id = token.sub;
      return session;
    },
  },
};

export const { handlers, auth, signIn, signOut } = NextAuth(authConfig);
```

Note the dummy hash: when no user is found we still run `verifyPassword` against a fixed garbage hash so the response time does not leak existence. This is a known mitigation pattern (used by GitLab, others).

```ts
"use server";

import { redirect } from "next/navigation";
import { headers } from "next/headers";
import { prisma } from "@/server/db";
import { readEnv } from "@/server/env";
import { hashPassword, verifyPassword } from "@/server/auth/password";
import { getSimpleSession } from "@/server/auth/iron";
import { check as rlCheck, recordFailure, recordSuccess } from "@/server/auth/rate-limit";
import { writeAuditLog } from "@/server/auth/audit";
import {
  InvalidCredentialsError,
  RateLimitedError,
} from "@/lib/errors";

export { requireUserId } from "@/server/auth";

const LOCAL_DEFAULT = "local-default";

async function clientIp(): Promise<string> {
  const h = await headers();
  return (h.get("x-forwarded-for") ?? "0.0.0.0").split(",")[0].trim();
}

export async function setSimplePassword(formData: FormData): Promise<void> {
  if (readEnv().AUTH_MODE !== "simple") {
    throw new Error("setSimplePassword only valid in AUTH_MODE=simple");
  }
  const password = String(formData.get("password") ?? "");
  const confirm = String(formData.get("confirm") ?? "");
  if (password.length < 8) throw new Error("Password too short");
  if (password !== confirm) throw new Error("Passwords do not match");

  const hash = await hashPassword(password);
  await prisma.user.update({ where: { id: LOCAL_DEFAULT }, data: { passwordHash: hash } });
  await writeAuditLog({ action: "password.set", userId: LOCAL_DEFAULT, ip: await clientIp() });

  // Auto sign-in after setting
  const session = await getSimpleSession();
  session.userId = LOCAL_DEFAULT;
  session.setAt = Date.now();
  await session.save();

  redirect("/");
}

export async function signInWithPassword(input: {
  email: string | null;
  password: string;
}): Promise<void> {
  const env = readEnv();
  const ip = await clientIp();
  const rl = rlCheck(ip);
  if (!rl.ok) {
    await writeAuditLog({ action: "login.ratelimit", ip });
    throw new RateLimitedError(rl.retryAfterSec);
  }

  if (env.AUTH_MODE === "simple") {
    const user = await prisma.user.findUnique({ where: { id: LOCAL_DEFAULT } });
    const ok = user?.passwordHash
      ? await verifyPassword(input.password, user.passwordHash)
      : false;
    if (!ok) {
      recordFailure(ip);
      await writeAuditLog({
        action: "login.fail", ip, userId: user?.id ?? null,
        meta: user?.passwordHash ? undefined : { reason: "no-password-set" },
      });
      throw new InvalidCredentialsError();
    }
    recordSuccess(ip);
    const session = await getSimpleSession();
    session.userId = user!.id;
    session.setAt = Date.now();
    await session.save();
    await writeAuditLog({ action: "login.success", ip, userId: user!.id });
    return;
  }

  if (env.AUTH_MODE === "full") {
    if (!input.email) throw new InvalidCredentialsError();
    const { signIn } = await import("@/server/auth/nextauth");
    // Auth.js authorize() runs its own audit + rate-limit. Here we just delegate.
    await signIn("credentials", { email: input.email, password: input.password, redirect: false });
    return;
  }

  throw new Error("signInWithPassword called when AUTH_MODE=none");
}

export async function signOut(): Promise<void> {
  const env = readEnv();
  const ip = await clientIp();
  let uid: string | null = null;
  if (env.AUTH_MODE === "simple") {
    const session = await getSimpleSession();
    uid = session.userId ?? null;
    await session.destroy();
  } else if (env.AUTH_MODE === "full") {
    const { signOut: naSignOut, auth } = await import("@/server/auth/nextauth");
    const session = await auth();
    uid = (session?.user as { id?: string } | undefined)?.id ?? null;
    await naSignOut({ redirect: false });
  }
  await writeAuditLog({ action: "logout", ip, userId: uid });
  redirect(env.AUTH_MODE === "none" ? "/" : "/login");
}
```

```ts
import { NextResponse, type NextRequest } from "next/server";

const PUBLIC = new Set(["/login", "/signup", "/api/auth"]);

function isPublic(path: string) {
  for (const p of PUBLIC) if (path === p || path.startsWith(p + "/")) return true;
  // assets
  if (path.startsWith("/_next") || path === "/favicon.ico") return true;
  return false;
}

export async function middleware(req: NextRequest) {
  const mode = process.env.AUTH_MODE ?? "none";
  if (mode === "none") return NextResponse.next();
  if (isPublic(req.nextUrl.pathname)) return NextResponse.next();

  // Cookie presence is a fast path - server actions still call requireUserId() for the strong check.
  const cookie = req.cookies.get("tdm_session");
  if (!cookie) {
    const url = req.nextUrl.clone();
    url.pathname = "/login";
    return NextResponse.redirect(url);
  }
  return NextResponse.next();
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
};
```

Cookie presence is a hint, not the source of truth: the cookie could be expired or forged. Server actions still call `requireUserId()` which calls into the real session machinery.

## Notes for the agent

**Phase 8 contract (do not break this):**
- Every server action under `src/server/actions/` calls `requireUserId()` as the first line. Reviewers should reject any PR that resolves `userId` from `formData`, the request body, or a query parameter.
- Import path: `@/server/auth` - keep it stable. Renaming the module is a breaking change for every later phase.
- `requireUserId()` returns `string`; never `string | null`. Throws `UnauthenticatedError` instead.
- `LlmCredential.userId` (Phase 8) is a foreign key into `User.id`. The HKDF subkey for AES-256-GCM uses `userId` as the salt (see `docs/security.md`). This relies on `userId` being a real, session-resolved string - never user input.

**`LoginAttempt` writer is a stretch goal:**
- Table is created here; current implementation logs to `AuditLog` only. If you want belt-and-suspenders, add `prisma.loginAttempt.create()` alongside each `writeAuditLog()` call in `signInWithPassword()` and the Auth.js `authorize()` callback.

**`getCurrentUserId()` is now internal:**
- It still exists for `requireUserId()` to call, but no future phase should import it directly. If a route legitimately allows anonymous access (none in v1), it should explicitly tolerate `null`.

**iron-session vs Auth.js for `simple` mode:**
- iron-session was chosen because `simple` mode has exactly one user, needs no User lookup for session resolution, no DB round-trip per request, keeps `next-auth` out of the bundle when unused, and the cookie shape is trivial. The `tdm_session` cookie name is shared between modes - safe because a deployment is one mode at a time and boot guard enforces it.

**Constant-time existence hiding:**
- Auth.js `authorize()` runs `verifyPassword(password, DUMMY_HASH)` even when the user is missing. This makes response time independent of user existence. The dummy hash is a real Argon2id PHC string with the same params. Do not short-circuit on missing user.

**Timing leak on `simple` mode setup state:**
- `signInWithPassword` short-circuits when `passwordHash` is null, leaking via timing whether setup has happened. Acceptable: that fact also leaks via the `/signup` redirect, and `simple` mode is single-user.

**Out of scope this phase (Phase 14 lands these):**
- Strict CSP and HSTS headers (cookie attrs are tested here; the rest is Phase 14)
- CSRF tokens on mutations (Next.js same-origin + Auth.js cover most cases)
- Multi-instance rate limiter (Redis) - in-memory bucket is single-instance-only
- Audit log for non-auth mutations
- Email-verification + password-reset flows (requires SMTP)
- Account-linking UI for OAuth (Phase 12)
- Per-user account deletion (Phase 14)

When all boxes in the acceptance criteria are checked and `pnpm test && pnpm build` are green for all four config matrices, Phase 7 is done. Move to Phase 8 (settings + LLM providers + content fetcher): write `phase-8-settings-llm.md` immediately before starting it. The first task in Phase 8 will be the AES-256-GCM crypto module that depends on `requireUserId()` from this phase.
