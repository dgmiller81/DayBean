# Phase 8 — Settings, LLM Providers, Crypto, SSRF-Guarded Fetch, and Manual Refresh Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the user-facing Settings modal (Profile / LLM Provider / Theme / Job & Interests), encrypt user-supplied LLM API keys at rest with AES-256-GCM + HKDF-derived per-user subkeys, ship multi-provider LLM adapters (OpenAI, Anthropic, LM Studio via OpenAI-compat) behind a uniform `generateDailyContent()` interface, ship an SSRF-guarded server-side web fetcher (Readability + jsdom) for article extraction, and wire the "Refresh today's content" button that Phase 6 stubbed so it actually generates a new `DailyContent` row. Per-user manual refresh is rate-limited to 3/day; web fetch is rate-limited to 30/user/hour.

**Deploy target this phase:** Both `local` and `railway`. The crypto contract, SSRF guard, and rate limits behave identically in both modes; the SSRF guard's "is `http:` allowed" decision is the only env-driven branch.

**Phase 8 is security-critical.** It owns the trust boundary between the user's plaintext API key entering the server, the encrypted-at-rest storage, the per-request decryption, and the egress-only web fetcher. Every security decision below has a written rationale (why HKDF and not PBKDF2; why 12-byte IV; why 8-second timeout; why size cap at 1 MB). When in doubt during implementation, re-read the rationale rather than improvise.

This is the security-critical heart of Phase 8. Write the test first, watch it fail, then implement.

### Design rationale (lock these decisions in)

**Why AES-256-GCM and not AES-CBC + HMAC?** GCM is authenticated encryption in a single primitive — tampering with ciphertext or the IV makes `decipher.final()` throw. CBC+HMAC requires manual MAC handling and is famously easy to get wrong (constant-time compare, MAC-then-encrypt vs encrypt-then-MAC ordering).

**Why HKDF and not PBKDF2?** The `APP_ENCRYPTION_KEY` env var is *already* a high-entropy 32-byte key — it doesn't need to be stretched from a low-entropy password, which is what PBKDF2 is for. HKDF is the right primitive when you have a master key and want domain-separated subkeys for many users (per RFC 5869). It is fast (no work-factor cost) and the per-user salt (`userId`) plus the `info='llm-cred'` tag means a leaked subkey for one user does not compromise others, and a future "encrypt session tokens" subsystem reusing the same master key (with `info='session'`) gets cryptographically distinct keys for free.

**Why 12-byte IV?** GCM's standard nonce length per NIST SP 800-38D §8.2.1 is 96 bits (12 bytes). Random 12-byte IVs collide at 2^48 encryptions per key — astronomically beyond any plausible per-user key budget. Going larger forces internal re-hashing inside GCM and adds bytes for no benefit.

**Why store `iv | ciphertext | tag` concatenated and base64-encoded?** Single-column storage is easier to migrate, audit, and rotate than three columns. The output format is base64 to keep it printable in DB tools. Decoding is trivial: `iv = bytes[0..12]`, `tag = bytes[bytes.length-16..]`, `ciphertext = bytes[12..bytes.length-16]`.

**Why `userId` as HKDF salt?** The salt's job in HKDF is domain separation, not secrecy. `userId` is non-secret, unique-per-user, and stable for the lifetime of the credential — exactly what HKDF wants. If we used a random per-row salt we'd have to store it; using `userId` lets us re-derive the subkey from `(masterKey, userId)` alone.

- [ ] **Step 2: Implement `src/server/crypto.ts`**

```ts
import { createCipheriv, createDecipheriv, hkdfSync, randomBytes } from "node:crypto";

const IV_LEN = 12;          // 96 bits — NIST SP 800-38D recommended
const TAG_LEN = 16;         // 128 bits — GCM authentication tag
const KEY_LEN = 32;         // 256-bit subkey
const HKDF_INFO = Buffer.from("llm-cred");

function masterKey(): Buffer {
  const raw = process.env.APP_ENCRYPTION_KEY;
  if (!raw) throw new Error("APP_ENCRYPTION_KEY missing");
  const key = Buffer.from(raw, "base64");
  if (key.length !== KEY_LEN) {
    throw new Error("APP_ENCRYPTION_KEY must decode to 32 bytes");
  }
  return key;
}

function subkey(userId: string): Buffer {
  // HKDF(master, salt=userId, info='llm-cred', length=32)
  const out = hkdfSync("sha256", masterKey(), Buffer.from(userId, "utf8"), HKDF_INFO, KEY_LEN);
  return Buffer.from(out);
}

export function encrypt(plaintext: string, userId: string): string {
  const iv = randomBytes(IV_LEN);
  const key = subkey(userId);
  const cipher = createCipheriv("aes-256-gcm", key, iv);
  const ct = Buffer.concat([cipher.update(plaintext, "utf8"), cipher.final()]);
  const tag = cipher.getAuthTag();
  return Buffer.concat([iv, ct, tag]).toString("base64");
}

export function decrypt(encoded: string, userId: string): string {
  const buf = Buffer.from(encoded, "base64");
  if (buf.length < IV_LEN + TAG_LEN + 1) throw new Error("ciphertext too short");
  const iv = buf.subarray(0, IV_LEN);
  const tag = buf.subarray(buf.length - TAG_LEN);
  const ct = buf.subarray(IV_LEN, buf.length - TAG_LEN);
  const key = subkey(userId);
  const decipher = createDecipheriv("aes-256-gcm", key, iv);
  decipher.setAuthTag(tag);
  const pt = Buffer.concat([decipher.update(ct), decipher.final()]);
  return pt.toString("utf8");
}

export function last4Of(plaintext: string): string {
  return plaintext.slice(-4);
}
```

Note: this module imports nothing from `log.ts` — the plaintext never gets the chance to be logged inside crypto. The log-spy test passes by construction.

```ts
import pino from "pino";

const REDACT_PATHS = [
  "*.apiKey",
  "*.encryptedKey",
  "*.password",
  "*.passwordHash",
  "*.authorization",
  "*.cookie",
  "apiKey",
  "encryptedKey",
  "password",
  "authorization",
];

export const logger = pino({
  level: process.env.LOG_LEVEL ?? (process.env.NODE_ENV === "production" ? "info" : "debug"),
  redact: { paths: REDACT_PATHS, censor: "[REDACTED]" },
});

/**
 * Returns a shallow-cloned object with known-sensitive keys redacted.
 * Use this in any log site that takes a free-form payload (e.g., a server
 * action's input). Nested objects are NOT recursed — keep the shapes flat.
 */
export function logSafe<T extends Record<string, unknown>>(obj: T): Record<string, unknown> {
  const out: Record<string, unknown> = { ...obj };
  for (const k of Object.keys(out)) {
    if (REDACT_PATHS.some((p) => p.toLowerCase().endsWith(k.toLowerCase()))) {
      out[k] = "[REDACTED]";
    }
  }
  return out;
}
```

The `pino` `redact` option handles structured logs; `logSafe()` is the escape hatch for any path that wants to log a manually-built object. Nothing in this phase calls `logger` with a raw plaintext key — but the test exists so that any future regression is caught.

### Design rationale

**Why in-memory and not Redis?** Rate limits in this phase are per-user, not per-IP, and the user budgets (3 refreshes/day, 30 fetches/hour) are tiny enough that the per-user state fits easily in process memory. On Railway we run a single instance per deployment, so a process-local map is correct. If we ever scale horizontally (which is out-of-scope for v1; see roadmap §"De-scoped"), this module is the single point of replacement.

**Why per-user and not per-IP?** API costs and SSRF abuse correlate with the user, not the IP. Per-IP rate limits exist for the auth flow (Phase 7); this is a separate axis.

**Why a plain object and not a sliding-window algorithm?** Daily refresh is a calendar-day budget — at midnight local, the count resets. The fetch limit is rolling-hour. Both are simple `lastResetAt + count` structures.

```ts
const MAX_REFRESH_PER_DAY = 3;
const MAX_FETCH_PER_HOUR = 30;
const ONE_HOUR_MS = 60 * 60 * 1000;

export class RateLimitError extends Error {
  constructor(public readonly kind: "refresh" | "fetch", public readonly retryAfterSeconds: number) {
    super(`rate-limited:${kind}`);
    this.name = "RateLimitError";
  }
}

const refreshCounts = new Map<string, number>(); // key = `${userId}:${iso}`
const fetchBuckets = new Map<string, { count: number; resetAt: number }>(); // key = userId

export function assertRefreshAllowed(userId: string, iso: string): void {
  const key = `${userId}:${iso}`;
  const next = (refreshCounts.get(key) ?? 0) + 1;
  if (next > MAX_REFRESH_PER_DAY) {
    // Until end of iso day in UTC. Phase 11 will revisit per-user TZ.
    const tomorrowMidnightUtc = Date.UTC(
      Number(iso.slice(0, 4)),
      Number(iso.slice(5, 7)) - 1,
      Number(iso.slice(8, 10)) + 1,
    );
    const retry = Math.max(1, Math.floor((tomorrowMidnightUtc - Date.now()) / 1000));
    throw new RateLimitError("refresh", retry);
  }
  refreshCounts.set(key, next);
}

export function assertFetchAllowed(userId: string): void {
  const now = Date.now();
  const cur = fetchBuckets.get(userId);
  if (!cur || cur.resetAt <= now) {
    fetchBuckets.set(userId, { count: 1, resetAt: now + ONE_HOUR_MS });
    return;
  }
  if (cur.count + 1 > MAX_FETCH_PER_HOUR) {
    throw new RateLimitError("fetch", Math.max(1, Math.floor((cur.resetAt - now) / 1000)));
  }
  cur.count += 1;
}

export function __resetForTests(): void {
  refreshCounts.clear();
  fetchBuckets.clear();
}
```

### Design rationale

**Why DNS-resolve before fetch?** The naive check (`new URL(u).hostname` against a blocklist) is bypassed by `http://attacker-controlled-dns-record.example.com` resolving to `127.0.0.1` or `169.254.169.254` (cloud metadata). We resolve first, validate every returned address against the private/loopback/link-local blocklist, then fetch *with the validated IP* (or, alternatively, fetch the host but verify after — see implementation note below). The implementation uses Node's `dns.lookup` and rejects if any A/AAAA record is private.

**Why re-validate every redirect?** A 302 from a public host to `http://169.254.169.254/latest/meta-data` is the canonical SSRF pivot. We disable automatic redirects (`redirect: 'manual'`), inspect the `Location` header, re-run the SSRF check on the new URL, and chase up to 3 hops total.

**Why 8-second timeout?** Long enough for a slow-but-legit news site (P95 well under 5s); short enough that 30 of these per hour can't occupy a worker forever. AbortController + setTimeout, cleared in `finally`.

**Why 1 MB cap?** Article HTML sits comfortably in the 200–500 KB range. 1 MB doubles that for safety while killing the "10 GB chunked-encoding response" attack. We enforce by streaming and counting bytes — not by trusting `Content-Length` (which an attacker controls).

**Why text/html only?** The fetcher's job is to feed Readability. PDFs, video, and binary blobs would be routed here only by an attacker trying to use the server as a generic exfil channel. Strict content-type allow-list closes that.

**Why http allowed in `local` mode?** Local dev tooling (LM Studio at `http://localhost:1234`) and self-hosted RSS readers. Production-ish (`railway`) refuses anything but `https:`.

```ts
import { lookup as dnsLookup } from "node:dns/promises";
import { isIP } from "node:net";
import { JSDOM } from "jsdom";
import { Readability } from "@mozilla/readability";
import { assertFetchAllowed } from "./rate-limit";
import { logger } from "./log";

const TIMEOUT_MS = 8_000;
const MAX_BYTES = 1024 * 1024;
const MAX_REDIRECTS = 3;
const ALLOWED_CONTENT_TYPES = ["text/html"];

// Test-injection seam — production code never sets this.
let dnsLookupImpl: ((host: string) => Promise<string>) | undefined;
export function __setDnsLookupForTests(fn: ((host: string) => Promise<string>) | undefined) {
  dnsLookupImpl = fn;
}

async function resolveHost(host: string): Promise<string> {
  if (dnsLookupImpl) return dnsLookupImpl(host);
  const r = await dnsLookup(host, { all: false, verbatim: true });
  return r.address;
}

function isPrivateAddress(addr: string): boolean {
  // IPv4
  if (isIP(addr) === 4) {
    const [a, b] = addr.split(".").map(Number);
    if (a === 10) return true;                              // 10.0.0.0/8
    if (a === 127) return true;                             // 127.0.0.0/8 loopback
    if (a === 169 && b === 254) return true;                // 169.254.0.0/16 link-local
    if (a === 172 && b >= 16 && b <= 31) return true;       // 172.16.0.0/12
    if (a === 192 && b === 168) return true;                // 192.168.0.0/16
    if (a === 0) return true;                               // 0.0.0.0/8
    if (a === 100 && b >= 64 && b <= 127) return true;      // 100.64.0.0/10 CGNAT
    if (a >= 224) return true;                              // multicast + reserved
    return false;
  }
  // IPv6 — normalize to lowercase first
  const v6 = addr.toLowerCase();
  if (v6 === "::1") return true;                            // loopback
  if (v6 === "::") return true;                             // unspecified
  if (v6.startsWith("fe80:")) return true;                  // link-local
  if (v6.startsWith("fc") || v6.startsWith("fd")) return true; // ULA fc00::/7
  if (v6.startsWith("ff")) return true;                     // multicast
  // ::ffff:a.b.c.d (IPv4-mapped) — re-check the IPv4 portion
  const m = v6.match(/^::ffff:([0-9.]+)$/);
  if (m && isIP(m[1]) === 4) return isPrivateAddress(m[1]);
  return false;
}

function assertSchemeAllowed(u: URL): void {
  if (u.protocol === "https:") return;
  if (u.protocol === "http:" && process.env.DEPLOY_TARGET === "local") return;
  throw new Error(`fetch refused: scheme ${u.protocol}`);
}

async function assertHostPublic(host: string): Promise<void> {
  const addr = isIP(host) ? host : await resolveHost(host);
  if (isPrivateAddress(addr)) {
    throw new Error(`fetch refused: host resolves to private/loopback/link-local address (${addr})`);
  }
}

async function readBodyCapped(res: Response): Promise<string> {
  const reader = res.body?.getReader();
  if (!reader) throw new Error("fetch refused: no response body");
  const decoder = new TextDecoder("utf-8");
  let total = 0;
  let out = "";
  for (;;) {
    const { value, done } = await reader.read();
    if (done) break;
    total += value.byteLength;
    if (total > MAX_BYTES) {
      try { await reader.cancel(); } catch { /* swallow */ }
      throw new Error("fetch refused: body too large");
    }
    out += decoder.decode(value, { stream: true });
  }
  out += decoder.decode();
  return out;
}

export interface FetchedArticle {
  title: string;
  text: string;
  url: string;
}

/**
 * Server-only article fetcher with SSRF guard.
 *
 * Per security.md §"SSRF Guard for Web Fetch":
 *  - https only (or http when DEPLOY_TARGET=local)
 *  - DNS-resolved IP must not be private / loopback / link-local
 *  - Content-Type must be text/html
 *  - Body capped at 1 MB
 *  - Wall-clock timeout 8s
 *  - Up to 3 redirects, each re-validated
 *  - Per-user rate-limited (30/hour)
 */
export async function fetchArticle(rawUrl: string, userId: string): Promise<FetchedArticle> {
  assertFetchAllowed(userId);

  let currentUrl = new URL(rawUrl);
  let body: string | null = null;
  let finalUrl = currentUrl;

  for (let hop = 0; hop <= MAX_REDIRECTS; hop++) {
    assertSchemeAllowed(currentUrl);
    await assertHostPublic(currentUrl.hostname);

    const ctrl = new AbortController();
    const timer = setTimeout(() => ctrl.abort(), TIMEOUT_MS);
    let res: Response;
    try {
      res = await fetch(currentUrl, {
        method: "GET",
        redirect: "manual",
        signal: ctrl.signal,
        headers: { "user-agent": "TheDailyMind/1.0", accept: "text/html" },
      });
    } catch (err) {
      const msg = (err as Error).message;
      throw new Error(msg.match(/abort/i) ? "fetch refused: timeout" : `fetch refused: ${msg}`);
    } finally {
      clearTimeout(timer);
    }

    if (res.status >= 300 && res.status < 400) {
      const loc = res.headers.get("location");
      if (!loc) throw new Error("fetch refused: redirect without Location");
      if (hop === MAX_REDIRECTS) throw new Error("fetch refused: too many redirects");
      currentUrl = new URL(loc, currentUrl);
      continue;
    }

    if (!res.ok) throw new Error(`fetch refused: status ${res.status}`);

    const ct = (res.headers.get("content-type") ?? "").split(";")[0].trim().toLowerCase();
    if (!ALLOWED_CONTENT_TYPES.includes(ct)) {
      throw new Error(`fetch refused: content-type ${ct}`);
    }

    body = await readBodyCapped(res);
    finalUrl = currentUrl;
    break;
  }

  if (body === null) throw new Error("fetch refused: no body after redirects");

  const dom = new JSDOM(body, { url: finalUrl.href });
  const article = new Readability(dom.window.document).parse();
  if (!article || !article.textContent) {
    throw new Error("fetch refused: readability could not extract article");
  }

  logger.info({ url: finalUrl.href, bytes: body.length }, "article fetched");

  return {
    title: (article.title ?? finalUrl.hostname).trim(),
    text: article.textContent.trim(),
    url: finalUrl.href,
  };
}
```

```ts
Output STRICT JSON matching this shape (no markdown, no commentary):

{
  "date": "YYYY-MM-DD",
  "subhead": "<one short evocative line>",

  "business": {

  },
  "personal": { "headline": "...", "motivation": { "text": "...", "author": "..." }, "articles": [...] }
}

Constraints:
- All "url" fields must be https URLs the user could click.
- Tone is warm, calm, bookish. No emojis.
- Bias topic selection by the user's role and content interests.
- Recent journal THEMES (not text) may bias the prayer / opening reflection.`;

export function buildUserPrompt(ctx: LlmContext): string {
  const lines = [
    `Today is ${ctx.iso}.`,
    ctx.name ? `User name: ${ctx.name}.` : "",
    ctx.jobTitle ? `Role: ${ctx.jobTitle}.` : "",
    ctx.contentInterests.length ? `Content interests: ${ctx.contentInterests.join(", ")}.` : "",
    ctx.recentJournalThemes.length
      ? `Recent journal themes (not the actual text — use only as soft bias): ${ctx.recentJournalThemes.join(", ")}.`
      : "",
    "",
    "Generate today's DAILY_CONTENT JSON now.",
  ].filter(Boolean);
  return lines.join("\n");
}
```

## Task 7: Refresh orchestrator

**Files:**
- Create: `src/server/llm/refresh.ts`

The orchestrator is the only path that mutates `DailyContent`. It:
1. Calls `requireUserId()` (Phase 7) at the boundary action — `refresh.ts` itself takes a `userId` it can trust.
2. Asserts the per-day rate limit.
3. Loads the user's pref (job, interests, name) and the configured `LlmCredential` row.
4. Loads recent journal **theme names** (Phase 3 already exposes a `recentJournalThemes(userId, days)` helper) — never raw text.
5. Calls the adapter.
6. Validates with `DailyContentSchema` (the adapter does this too — defense in depth).
7. Calls Phase 6's `setDailyContent`.
8. Writes a `RefreshLog` row.
9. Returns `{ ok: true }` or a structured error.

```ts
import { logger } from "@/server/log";
import { assertRefreshAllowed, RateLimitError } from "@/server/rate-limit";
import { DailyContentSchema } from "@/types";
import { setDailyContent } from "@/server/actions/daily-content"; // Phase 6
import { recentJournalThemes } from "@/server/journal-themes";   // Phase 3
import { getAdapter, type Provider } from "./index";
import type { LlmContext } from "./types";

export type RefreshSource = "manual" | "cron" | "cold-start";

export type RefreshResult =
  | { ok: true; iso: string }
  | { ok: false; code: "rate-limited"; retryAfterSeconds: number }
  | { ok: false; code: "no-credential" }
  | { ok: false; code: "provider-error"; message: string }
  | { ok: false; code: "validation-error"; message: string };

export async function refreshDailyContent(
  userId: string,
  iso: string,
  source: RefreshSource,
): Promise<RefreshResult> {
  // Manual refresh is the only one subject to the daily cap.
  // Cron/cold-start are scheduler-driven, capped elsewhere (Phase 10).
  if (source === "manual") {
    try {
      assertRefreshAllowed(userId, iso);
    } catch (e) {
      const ra = e instanceof RateLimitError ? e.retryAfterSeconds : 0;
      await prisma.refreshLog.create({
        data: { userId, iso, source, status: "rate-limited", errorCode: "rate-limited" },
      });
      return { ok: false, code: "rate-limited", retryAfterSeconds: ra };
    }
  }

  const cred = await prisma.llmCredential.findFirst({ where: { userId } });
  if (!cred) {
    await prisma.refreshLog.create({
      data: { userId, iso, source, status: "no-credential", errorCode: "no-credential" },
    });
    return { ok: false, code: "no-credential" };
  }

  const pref = await prisma.pref.findUnique({ where: { userId } });
  const user = await prisma.user.findUnique({ where: { id: userId }, select: { name: true } });

  const ctx: LlmContext = {
    userId,
    iso,
    name: user?.name ?? null,
    jobTitle: pref?.jobTitle ?? null,
    contentInterests: pref?.contentInterests ? JSON.parse(pref.contentInterests) : [],
    recentJournalThemes: await recentJournalThemes(userId, 7),
  };

  const log = await prisma.refreshLog.create({
    data: { userId, iso, source, status: "ok", errorCode: null },
  });

  try {
    const adapter = getAdapter(cred.provider as Provider);
    const content = await adapter.generateDailyContent(ctx);
    const validated = DailyContentSchema.parse(content); // belt + suspenders
    await setDailyContent(userId, iso, validated);
    await prisma.refreshLog.update({
      where: { id: log.id },
      data: { status: "ok", finishedAt: new Date() },
    });
    logger.info({ userId, iso, source, provider: cred.provider }, "refresh ok");
    return { ok: true, iso };
  } catch (e) {
    const msg = (e as Error).message;
    const code = msg.includes("ZodError") || /schema|validation/i.test(msg)
      ? "validation-error"
      : "provider-error";
    await prisma.refreshLog.update({
      where: { id: log.id },
      data: { status: code, errorCode: code, errorDetail: msg.slice(0, 1000), finishedAt: new Date() },
    });
    logger.warn({ userId, iso, source, code }, "refresh failed");
    return { ok: false, code, message: msg };
  }
}
```

```ts
const ProviderEnum = z.enum(["openai", "anthropic", "lmstudio"]);

const SaveLlmInput = z.object({
  provider: ProviderEnum,
  apiKey: z.string().min(8).max(512), // sanity bounds — real keys are ~40-200 chars
  model: z.string().min(1).max(200),
});

export type RedactedCredential = { provider: Provider; last4: string; model: string };

export async function getSettings() {
  const userId = await requireUserId();
  const [user, pref, creds] = await Promise.all([
    prisma.user.findUnique({ where: { id: userId }, select: { name: true, email: true } }),
    prisma.pref.findUnique({ where: { userId } }),
    prisma.llmCredential.findMany({
      where: { userId },
      select: { provider: true, last4: true, model: true },
    }),
  ]);
  return {
    profile: { name: user?.name ?? "", email: user?.email ?? "", bio: pref?.bio ?? "" },
    theme: pref?.theme ?? "light",
    job: { jobTitle: pref?.jobTitle ?? "", contentInterests: pref?.contentInterests ? JSON.parse(pref.contentInterests) : [] },
    credentials: creds as RedactedCredential[], // NO encryptedKey — projection guarantees this
  };
}

export async function saveProfile(input: { name: string; email: string; bio: string }) {
  const userId = await requireUserId();
  await prisma.user.update({ where: { id: userId }, data: { name: input.name, email: input.email } });
  await prisma.pref.upsert({
    where: { userId },
    update: { bio: input.bio },
    create: { userId, bio: input.bio },
  });
  return { ok: true as const };
}

export async function saveLlmCredential(rawInput: unknown): Promise<RedactedCredential> {
  const userId = await requireUserId();
  const input = SaveLlmInput.parse(rawInput);
  const encryptedKey = encrypt(input.apiKey, userId);
  const last4 = last4Of(input.apiKey);
  // input.apiKey now goes out of scope — the only persisted form is encryptedKey.
  await prisma.llmCredential.upsert({
    where: { userId_provider: { userId, provider: input.provider } },
    update: { encryptedKey, last4, model: input.model },
    create: { userId, provider: input.provider, encryptedKey, last4, model: input.model },
  });
  logger.info({ userId, provider: input.provider }, "llm credential saved");
  return { provider: input.provider, last4, model: input.model };
}

export async function testLlmConnection(provider: Provider) {
  const userId = await requireUserId();
  const adapter = getAdapter(provider);
  return adapter.testConnection({ userId });
}

export async function saveTheme(theme: "light" | "dark") {
  const userId = await requireUserId();
  await prisma.pref.upsert({
    where: { userId },
    update: { theme },
    create: { userId, theme },
  });
  return { ok: true as const };
}

export async function saveJobInterests(input: { jobTitle: string; contentInterests: string[] }) {
  const userId = await requireUserId();

    create: { userId, jobTitle: input.jobTitle, contentInterests: JSON.stringify(input.contentInterests) },
  });
  return { ok: true as const };
}
```

## Phase 8 Acceptance Criteria

Mapped to the security doc and master roadmap design calls.

**Crypto (security.md §"API Key Encryption"):**
- [ ] `encrypt(plaintext, userId)` returns base64(`iv | ciphertext | tag`)
- [ ] AES-256-GCM, 12-byte IV, 16-byte tag, IV randomized per call
- [ ] HKDF-SHA256 derives a 32-byte subkey with `salt=userId`, `info='llm-cred'`
- [ ] Decrypt with the wrong `userId` throws (HKDF domain separation)

## Notes for the Agent

**Contract Phase 9 (Onboarding) inherits:**

Phase 9's first-run wizard ends by calling `saveLlmCredential` with the user's chosen provider — exactly the same server action this phase ships. The wizard MUST NOT bypass the encrypted path; if it ever needs to seed a credential without prompting (e.g., LM Studio with no real key), it still goes through `saveLlmCredential({ provider: 'lmstudio', apiKey: 'lm-studio-no-key-needed', model })`. The "fake key" is the placeholder; the encryption + storage path remains identical.

**Contract Phase 10 (Scheduler) inherits:**

The scheduler's per-user job calls `refreshDailyContent(userId, iso, 'cron')` — the orchestrator already exists. Phase 10's only schema work is none — `RefreshLog` already exists. Phase 10's only behavioral additions are:
1. The cron handler (`/api/cron/refresh-all`) gated by `CRON_SECRET`
2. Idempotency: before calling the orchestrator, query `RefreshLog` for `(userId, iso)` with `status='ok'`; if found, skip
3. Cold-start scan: on app boot, list users whose latest `ok` `RefreshLog` predates today and trigger a `'cold-start'` refresh

**Schema Phase 10 reads (already created in Phase 8):**

```
LlmCredential (userId, provider, encryptedKey, last4, model, createdAt, updatedAt)
  unique (userId, provider)

RefreshLog   (userId, iso, source, status, tokensUsed, errorCode, errorDetail, startedAt, finishedAt)
  source ∈ {'manual','cron','cold-start'}
  status ∈ {'ok','rate-limited','no-credential','provider-error','validation-error'}
  index (userId, iso); index (userId, startedAt)
```

**Things deliberately deferred:**

- Multi-instance rate limiting — Phase 8 uses an in-memory map. If we ever scale Railway horizontally (out of scope per roadmap §"De-scoped"), swap this module for Redis or a DB-backed `RateLimitBucket` table.
- Encryption-key rotation — security.md §"Incident Response" describes the manual rotation procedure (decrypt-with-old, re-encrypt-with-new across all `LlmCredential` rows). Not automated in v1.
- Streaming generation — `generateText` is synchronous from the user's perspective. Streaming UX (token-by-token reveal) is a Phase-12+ polish item.
- Tokens-used accounting — `RefreshLog.tokensUsed` defaults to 0 in Phase 8. Phase 10 wires real numbers from the AI SDK's response metadata.
- Friends & Family LLM context — Phase 12's stub does not yet feed the prompt. The `buildUserPrompt` function is structured so that adding a `friendsAndFamily` field is a one-line change.

**Spec gaps surfaced during planning (informational — none block this phase):**

1. The mockup spec defers daily-content generation to a human; Phase 8 introduces an LLM-generated path. The shape contract from spec §5 is preserved verbatim and policed by `DailyContentSchema` (Phase 3). When the spec and the schema disagree, the schema wins because it's the runtime gate.
2. The security doc lists `metadata.google.internal` by name. We rely on DNS resolution returning `169.254.169.254` (which the link-local check rejects) rather than maintaining a hostname blocklist — this is a stronger guarantee.
3. Per-user time zones for the daily-refresh budget reset are not addressed in v1; the implementation uses UTC midnight. Phase 11 (calendar nav) is the natural place to revisit this.
